# Prior-Art Boundary

DT/FDS does not claim priority over the existence of the following concepts.
The claimed contribution is the dependency chain and finite-system architecture connecting them.

## Summary

DT/FDS is not a replacement for prior theories.
It is a finite-system framework that organizes boundary maintenance, capacity deficit, approximation, irreversible update, pruning, externalization, collapse, and invariant-supported persistence.

## Contribution Claim

The contribution of DT/FDS is not any single imported concept.

The contribution is the following dependency architecture:

```
distinction
→ boundary
→ finite capacity
→ capacity deficit
→ approximation
→ corrective complexity
→ irreversible update
→ dissipative / resource pressure
→ pruning / externalization / task relaxation / collapse
→ invariant-supported persistence
```

## Evaluation Rule

A critique should specify which layer it targets:

1. formal core,
2. physical bridge,
3. operational AI criterion,
4. domain mapping,
5. high-risk interpretive extension.
