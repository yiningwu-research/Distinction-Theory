# DT/FDS Reproducibility and Auditability

This repository distinguishes four types of claims:

1. **Formal definitions and conditional theorems** — reproducibility is logical (proof review).
2. **Physical bridge claims** — reproducibility requires physical or mathematical verification.
3. **Simulation-supported claims** — reproducibility requires source code, parameters, and execution.
4. **Domain bridge claims** — reproducibility requires domain-specific methods.

## Simulation Reproducibility

For simulation-supported claims, the repository should include:
- source code;
- parameter files;
- figure-generation scripts;
- random seeds where applicable;
- generated CSV tables;
- PDF figures;
- a short reproduction guide.

## Pre-Registered Predictions

For pre-registered predictions, the repository should include:
- prediction statement;
- publication date;
- frozen source file;
- DOI or OSF registration;
- failure condition;
- post-result update protocol.
